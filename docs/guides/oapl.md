# Optimal Advantage-Based Policy Optimization

Optimal Advantage-Based Policy Optimization with Lagged Inference Policy (OAPL) is an off-policy RL objective for LLM post-training from [LLMs Can Learn to Reason Via Off-Policy RL](https://arxiv.org/abs/2602.19362). It uses the same rollout, reward, and policy training infrastructure as GRPO, but replaces the clipped policy-gradient objective with a sequence-level squared-regression objective against an optimal-advantage target.

OAPL is useful when rollout samples are generated by an inference policy that can lag behind the trainer. Instead of correcting this mismatch with importance-sampling ratios, OAPL explicitly treats the inference policy as the KL anchor.

## Objective

For each prompt `x`, OAPL samples a group of responses `y_i` from the lagged inference policy `pi_vllm`. It estimates the KL-regularized optimal value:

```text
V*(x) = beta1 * log(mean_i exp(r(x, y_i) / beta1))
```

The sequence-level advantage target is:

```text
A*(x, y_i) = r(x, y_i) - V*(x)
```

The policy loss regresses the current trainer policy's sequence log-ratio to that target:

```text
(beta2 * log(pi(y_i | x) / pi_vllm(y_i | x)) - A*(x, y_i))^2
```

In NeMo RL, `pi_vllm` is represented by the generation log probabilities attached to rollouts, while `pi` is represented by the current policy log probabilities computed during training.

## Configuration

Enable OAPL in a GRPO-style recipe by selecting both the OAPL loss and the OAPL advantage estimator:

```yaml
loss_fn:
  name: oapl
  reference_policy_kl_penalty: 0.0
  vstar_beta: 1.0
  policy_beta: 1.0e-3
  length_normalize_log_ratio: false
  sync_interval: 50

grpo:
  val_period: 50
  use_dynamic_sampling: false
  seq_logprob_error_threshold: null
  skip_reference_policy_logprobs_calculation: true
  adv_estimator:
    name: oapl
```

`vstar_beta` is the smoothing coefficient used in the log-sum-exp value estimate. `policy_beta` scales the sequence log-ratio term in the squared loss. `sync_interval` controls how many trainer steps elapse before the generation policy is refit from the trainer in the synchronous GRPO loop.

The OAPL paper reports math experiments with `vstar_beta=1`, `policy_beta=1e-3`, and `sync_interval=50`.

## Compatibility

OAPL validates these settings at startup:

| Setting | Required value |
|---|---|
| `loss_fn.name` and `grpo.adv_estimator.name` | both `oapl` |
| `loss_fn.reference_policy_kl_penalty` | `0.0` |
| `loss_fn.use_importance_sampling_correction` | `false` |
| `grpo.skip_reference_policy_logprobs_calculation` | `true` |
| `grpo.use_dynamic_sampling` | `false` |
| `grpo.seq_logprob_error_threshold` | `null` |
| `grpo.val_period` | `0` or a multiple of `loss_fn.sync_interval` |

The validation cadence restriction keeps reported validation metrics aligned with generation-policy sync boundaries.

## Metrics

OAPL logs the scalar loss as `oapl_mse` and reports sequence-regression diagnostics including `oapl/residual_mean`, `oapl/residual_std`, `oapl/seq_log_ratio_mean`, and `oapl/target_mean`. It also emits train-vs-generation diagnostics used for policy-lag debugging: `token_mult_prob_error`, `gen_kl_error`, `policy_kl_error`, `js_divergence_error`, and `approx_entropy`.

## Notes

- OAPL does not require a reference-policy KL penalty; set `reference_policy_kl_penalty: 0.0` and skip reference logprob calculation.
- OAPL is sequence-level, so it optimizes the sum of response-token log-ratios by default. Set `length_normalize_log_ratio: true` only when intentionally experimenting with length-normalized variants.
- Reward shaping can still be configured through the existing GRPO/DAPO fields, but the OAPL objective itself does not use PPO clipping or importance-sampling correction.
